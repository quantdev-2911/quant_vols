SVI-JWK Boundary-Conditioned Arbitrage Transition Test

Test design specification — v2 — revised after internal code review of the v1 skeleton. Addresses the Head of MV's challenge that the existing §5.5 and §5.8 boundary tests do not link boundary proximity to arbitrage outcomes.

Prepared for document_2 File · Pre-Read
Executive summary

The existing boundary tests in document_1 §5.5 and document_2 §5.8 inspect how the smile shape looks when an SVI-JWK parameter is at or near its algebraic boundary. The Head of Validation's challenge is that this does not answer the validator's core question: at what distance from the calibrated mark does the first static-arbitrage violation appear, and along which parameter direction? This document specifies a test design — the Boundary-to-Arbitrage Transition (BAT) test — that answers this question directly, using the Durrleman $g$-function as the primary butterfly primitive and a monotonicity check on total implied variance for the calendar dimension.

Version 3 incorporates review corrections (changed for clarity): a non-raising parameter classification so the walker can evaluate surfaces up to the algebraic boundary and record outside-domain points as admissibility breaches; an explicit SVI-JWK → raw-SVI converter (document_1 eqs 4.10–4.15) with a post-rescaling $\kappa_t$ rescaling flag; the analytical Durrleman $g$-function as the primary butterfly metric alongside a numerical Breeden–Litzenberger cross-check; a calendar-spread check across tenors; an explicit materiality-thresholds structure that must be populated before the walker can produce pass/fail output; corrected beta-zero JWK inversion handling; explicit sigma-zero kink diagnostics; and an intermediate-tenor JWK interpolation helper for calendar testing (changed for clarity).

Scope excludes probability-of-occurrence and materiality-unit quantification (deferred, per validation team direction).

§ 01 — Why the prior boundary test is incomplete

From smile inspection to arbitrage transition

The document_1 §5.5 and document_2 §5.8 boundary tests push each SVI-JWK parameter to its analytical limit one at a time and report what the smile looks like. Both reach the same qualitative finding — the put wing or call wing jumps up when $\psi_t$ or the wing-slope parameters hit their boundary — and both close with a monitoring recommendation rather than a measurement. Three questions that the existing tests do not answer:

  1. whether a near-boundary or past-boundary parameter vector produces a static-arbitrage violation that could be exploited by a calendar or butterfly spread;
  2. which direction in parameter space produces the first material arbitrage and how far the current calibrated marks sit from that failure;
  3. whether the algebraic boundary of the SVI-JWK parameter space coincides with the arbitrage-free boundary — or whether the two differ, as Gatheral-Jacquier (2014) Theorem 4.2 suggests.

The proposed test is therefore not another generic butterfly or calendar check: it is a boundary-conditioned arbitrage transition test that links parameter-boundary proximity directly to surface-level arbitrage outcomes.

§ 02 — Recommended test structure

The Boundary-to-Arbitrage Transition (BAT) test

The test has three components. Each answers a distinct question and each is supported by closed-form or transparent numerical primitives in the companion Python module.

Table 1 · BAT test components
Component Question answered Why it is not duplicative of existing §5.11 or §5.18 tests
Boundary classification Is the parameter vector inside, at, or past the algebraic admissibility region? The existing validate-and-raise pattern cannot distinguish the three cases; the classifier returns a structured diagnostic.
First-arbitrage transition Along a chosen parameter direction, at what shock magnitude does the first material arbitrage appear? The existing arbitrage checks evaluate pass/fail at the calibrated mark. This test measures the distance to failure.
Calendar-spread check Does $w(k,T)$ remain monotonic in $T$ across the full dense moneyness grid, including between calibrated tenors? The existing check evaluates calendar non-arbitrage only at calibrated tenors; this extends to the full grid the MC pricer traverses.
§ 03 — Mathematical set-up

Primitives and boundary conditions

3.1 · Raw SVI total variance

For log-moneyness $k = \log(K/F)$, the raw-SVI total variance is

$$w(k) = a + b\left\{\rho(k - m) + \sqrt{(k - m)^2 + \sigma^2}\right\}$$

with algebraic admissibility region

$$b \geq 0, \qquad |\rho| \leq 1, \qquad \sigma \geq 0, \qquad a + b\sigma\sqrt{1 - \rho^2} \geq 0.$$

Note the inequalities are non-strict — this matters for the boundary-transition test, which must be able to evaluate surfaces at $|\rho| = 1$ and $\sigma = 0$ (the $\psi_t$-boundary collapse case).

3.2 · SVI-JWK → raw-SVI inversion

document_1 equations (4.10)–(4.15) define the inversion. The key quantities are

$$b = \frac{\sqrt{w_t}}{2}(c_t + p_t), \qquad \rho = \frac{c_t - p_t}{c_t + p_t}, \qquad \beta = \frac{c_t - p_t - 4\psi_t}{c_t + p_t}$$
$$\gamma = \frac{\sqrt{w_t}\,(c_t + p_t)}{4\left(2\sqrt{w_t}\,\kappa_t^{\text{raw}} + \psi_t^2\right)}, \qquad m = \beta(1 - \beta^2)\gamma, \qquad \sigma = \frac{\sqrt{1 - \beta^2}}{\beta}\,m$$

with $w_t = \nu_t \cdot T$ and $a = w_t - b\{-\rho m + \sqrt{m^2 + \sigma^2}\}$. The $\kappa_t$ input to the inversion must be in the pre-rescaling convention — the post-rescaling HyokaPara storage convention takes $\kappa_t^{\text{stored}} = \kappa_t^{\text{raw}} / \sqrt{T}$, so when consuming that storage the converter must multiply by $\sqrt{T}$ before using $\gamma$. The companion Python module exposes this via a kappa_rescaled flag on svi_jwk_to_raw_svi(…).

At $|\beta| = 1$ the inversion is degenerate: $m \to 0$, $\sigma \to 0$, $a \to w_t$. The converter handles this limit explicitly rather than producing NaN.

3.4 · Butterfly non-arbitrage: analytical Durrleman $g$-function

For any raw-SVI slice, the Durrleman function

$$g(k) = \left(1 - \frac{k\,w'(k)}{2\,w(k)}\right)^2 - \frac{w'(k)^2}{4}\left(\frac{1}{w(k)} + \frac{1}{4}\right) + \frac{w''(k)}{2}$$

is the standard analytical butterfly-arbitrage diagnostic under the usual smoothness, positive-total-variance and option-price boundary assumptions; non-negativity over the full log-moneyness domain is used as the regulator-grade condition for the implemented slice test (changed for clarity). For raw SVI the derivatives are closed-form:

$$w'(k) = b\rho + \frac{b(k - m)}{\sqrt{(k - m)^2 + \sigma^2}}, \qquad w''(k) = \frac{b\sigma^2}{\big((k - m)^2 + \sigma^2\big)^{3/2}}$$

The companion module uses these closed forms as the primary butterfly primitive. A numerical Breeden-Litzenberger density computed by finite difference on the call-price surface is retained as an independent cross-check — they should agree on sign everywhere.

Why the analytical g is the primary metric

Numerical $\partial^2 C / \partial K^2$ on an irregular strike grid is $O(h^2)$ accurate in the interior and $O(h)$ at the edges. A density that dips narrowly below zero across a few strikes can be smoothed away by finite differences, producing false negatives. The analytical $g$-function is therefore the primary metric for smooth positive-total-variance slices. At non-smooth boundary cases such as $\sigma=0$ exactly at $k=m$, the implementation flags undefined kink points rather than silently treating the slice as regular. For regulator-grade work, the analytical metric is the one the acceptance criterion should reference, with numerical density retained as a diagnostic cross-check (changed for clarity).

3.5 · Lee wing-slope bound

The asymptotic slopes of $w(k)$ as $|k| \to \infty$ are $b(1+\rho)$ on the right and $b(1-\rho)$ on the left. Roger Lee's (2004) moment formula requires both to lie in $[0, 2]$. A slope approaching 2 corresponds to an extreme-tail limiting case in which finite moments above the first may fail to exist; it should not be described as the risk-neutral first moment itself ceasing to exist under the standard forward-price no-arbitrage set-up (changed for clarity). This condition is an external constraint on the SVI parameter space — a parameter vector can be algebraically admissible yet violate Lee's bound at long tenors — and is evaluated alongside the Durrleman $g$-function in the slice metrics.

3.6 · Calendar-spread non-arbitrage

For a surface, the calendar-spread non-arbitrage condition is $\partial w(k, T)/\partial T \geq 0$ for every $k$. Both existing validations check this only at calibrated tenors — neither tests whether the condition holds between calibrated tenors under FO's non-standard $\sqrt{\nu_t}$ interpolation (document_2 §5.18.2). The BAT calendar component evaluates monotonicity of $w(k, T)$ on a dense $(k, T)$ grid that includes at least 10 interpolation points between every adjacent calibrated tenor pair.

§ 04 — Surface-output arbitrage metrics

What the test measures on each shocked slice

For each (shocked) parameter vector along a transition path, the module computes the following metrics on a dense moneyness grid (default 401 strikes, minimum 50):

Table 3 · Arbitrage metrics computed per slice
Metric Condition for non-arbitrage Interpretation
Analytical Durrleman $g$ $\min_k g(k) \geq 0$ Primary butterfly primitive (closed form)
Numerical BL density $\partial^2 C / \partial K^2 \geq 0$ Diagnostic finite-difference cross-check; expected to broadly agree with analytical $g$ on well-resolved interior grid regions, but not a pointwise acceptance criterion (changed for clarity)
Call-price monotonicity $\partial C / \partial K \leq 0$ Vertical spread non-arbitrage
Call-price lower slope $\partial C / \partial K \geq -\text{df}$ Upper bound on vertical spread payoff
Lee wing bounds $b(1 \pm \rho) \leq 2$ Lee asymptotic wing-slope compliance; extreme-tail moment sanity check (changed for clarity)
Calendar decrement $w(k, T_{i+1}) \geq w(k, T_i)$ for adjacent tenors Calendar-spread non-arbitrage on the dense grid

Breeden-Litzenberger density is obtained from

$$q(K, T) = \frac{1}{\text{df}} \cdot \frac{\partial^2 C(K, T)}{\partial K^2}$$

where df is the risk-free discount factor $e^{-rT}$. Dividend treatment is folded into the forward $F$ before the call pricer is invoked.

§ 05 — Boundary-to-arbitrage transition design

The shock walker

  1. Start from the production raw-SVI or SVI-JWK parameter vector for a given underlying and maturity. If SVI-JWK, convert to raw SVI using the inversion with the correct $\kappa_t$ rescaling flag.
  2. Classify the starting point as regular, at-algebraic-boundary, or outside-admissible-region. If outside, the validator must flag this as a finding before the walker runs.
  3. Choose a shock direction $\mathbf{d}$ in raw-SVI parameter space (a five-vector of perturbations $\Delta a, \Delta b, \Delta \rho, \Delta m, \Delta \sigma$). For a validator-grade sweep, include at minimum: $\rho \to \pm 1$, $\sigma \to 0$, and the $\psi_t$-boundary equivalent direction expressed in raw-SVI parameters.
  4. Generate an $\alpha$-grid of shock magnitudes from 0 (base case) through $\alpha_{\max}$ (the point that reaches or passes the boundary).
  5. For each $\alpha$, compute the shocked parameter vector and classify it. If the vector is outside the algebraic admissibility region, record this as the first admissibility breach and do not assume downstream surface metrics are meaningful. If the vector is regular or exactly at a permitted algebraic boundary, compute the slice metrics (changed for clarity).
  6. Apply the pre-approved materiality thresholds to each metric. The first $\alpha$ at which any threshold is exceeded is the first-breach alpha $\alpha^*$.
  7. Report $\alpha^*$, the breaching metric(s), the log-moneyness at which the breach was worst, and the distance from the calibrated mark to the shocked parameter vector in a chosen norm.
Materiality thresholds must be approved ex-ante

The walker requires a MaterialityThresholds object. If one is not supplied, the module returns results using floating-point-noise floors and flags the output with thresholds_were_defaulted=True. Results flagged this way should not be used for production sign-off — they are diagnostic only. The validation team must specify thresholds in writing (magnitude of $\min_k g$, integrated negative density, Lee-bound buffer, call-price monotonicity and slope tolerances) before the walker is used for regulator submission.

§ 06 — Tolerance approach

Acceptance thresholds

No universal numerical tolerance applies to all equity volatility surfaces. Thresholds must be pre-approved and referenced to numerical precision, bid-ask spread, strike spacing, option tick size, and portfolio materiality. The module does not enforce defaults; it forces the caller to supply them via MaterialityThresholds.

Table 4 · Tolerance principles by check
Check Recommended tolerance principle
Analytical Durrleman $g$ Theoretical strict inequality $g \geq 0$; material breach at $\min_k g < -\tau_g$ with $\tau_g$ expressing negligible density mass given the surface scale.
Numerical BL density Do not flag isolated single-strike noise. Require persistent adjacent negative nodes or integrated negative mass above approved threshold.
Lee wing slope Hard theoretical bound at 2. Numerical buffer permitted only for rounding; any buffer must be explicit and approved.
Call-price monotonicity / lower slope Reference to bid-ask spread and discount-factor / forward consistency. Breaches of the order of the bid-ask mid should be material.
Calendar decrement Floating-point floor only. Any persistent negative decrement is a real calendar-spread arbitrage signal.
§ 07 — Python implementation

Module and test suite

The companion Python module svi_boundary_arbitrage_checks_v2.py implements the primitives described above. The corresponding test suite test_svi_boundary_arbitrage_checks.py exercises all components on known-good and known-bad fixtures.

7.1 · Module contents

  • RawSVIParams, SVIJWKParams, SliceInputs, MaterialityThresholds — typed dataclasses for inputs.
  • classify_raw_svi_params — non-raising structured classification (regular / at-algebraic-boundary / outside-admissible-region).
  • svi_jwk_to_raw_svi — document_1 (4.10)–(4.15) inversion with kappa_rescaled flag.
  • raw_svi_total_variance_derivs, durrleman_g_analytical — closed-form total variance, first and second derivatives, and $g$-function.
  • black_call_from_total_variance — Black-76 pricer with explicit risk-free discount convention.
  • raw_svi_slice_arbitrage_metrics — per-slice diagnostic combining analytical $g$, numerical BL density, call monotonicity, lower-slope and Lee wing checks.
  • interpolate_jwk_slices_for_calendar_check — generates intermediate-tenor raw-SVI slices from JWK marks using the stated ATM-volatility and parameter interpolation convention (changed for clarity).
  • calendar_arbitrage_check — $w(k, T)$ monotonicity on the supplied dense tenor and moneyness grid across adjacent tenors.
  • first_detectable_breach_alpha — shock walker requiring MaterialityThresholds, reporting whether thresholds were defaulted.

7.2 · Quick-start example

import numpy as np
from svi_boundary_arbitrage_checks_v2 import (
    RawSVIParams, SVIJWKParams, SliceInputs, MaterialityThresholds,
    svi_jwk_to_raw_svi, raw_svi_slice_arbitrage_metrics,
    first_detectable_breach_alpha,
)

# 1. Convert production SVI-JWK marks to raw SVI
jwk = SVIJWKParams(nu_t=0.04, psi_t=-0.05, kappa_t=0.10, p_t=0.40, c_t=0.30)
params = svi_jwk_to_raw_svi(jwk, maturity=5.0, kappa_rescaled=True)

# 2. Define slice inputs (dense strike grid)
inputs = SliceInputs(
    strikes=np.linspace(60.0, 160.0, 401),
    forward=100.0, discount_factor=0.95, maturity=5.0,
)

# 3. Compute baseline slice metrics
metrics = raw_svi_slice_arbitrage_metrics(inputs, params)
print("Analytical min g =", metrics["min_g_analytical"])

# 4. Walk toward the rho = 1 boundary and find the first arbitrage
direction = RawSVIParams(a=0.0, b=0.0, rho=1.0, m=0.0, sigma=0.0)
alpha_grid = np.linspace(0.05, 1.8, 40)
thresholds = MaterialityThresholds(
    min_density_magnitude=1e-4,
    min_integrated_negative_density=1e-4,
    max_monotonicity_breach=1e-6,
    max_lower_slope_breach=1e-6,
    lee_bound_buffer=0.0,
)
result = first_detectable_breach_alpha(
    inputs, params, direction, alpha_grid, thresholds=thresholds,
)
print("First-breach alpha =", result["first_breach_alpha"])
print("Breach reasons     =", result["breach_reasons"])

7.3 · Test coverage

The enhanced test suite includes additional tests covering the beta-zero JWK conversion limit, sigma-zero kink diagnostics, and intermediate-tenor JWK interpolation, in addition to the earlier tests covering: classification at interior, at-boundary and past-boundary parameter vectors; SVI-JWK → raw-SVI round-trip at boundary and interior, with and without the $\kappa_t$ rescaling flag; Black-76 put-call parity and zero-variance intrinsic value; Durrleman $g$-function positivity on healthy surfaces, ability to go negative on adversarial surfaces, and sign agreement with the numerical density; calendar check pass on monotone surfaces and failure on inverted tenor order; shock walker reaching the algebraic boundary cleanly and reporting thresholds-were-defaulted when none are supplied; numpy 1.x / 2.x compatibility; and Lee moment bound activation.

§ 09 — Suggested validation conclusion

What the BAT test delivers to document_2

The Boundary-to-Arbitrage Transition test extends the previous boundary analysis from qualitative smile-shape inspection to quantitative measurement of boundary-induced static-arbitrage risk. For each calibrated slice, the test produces:

  • a classification of the parameter vector against the algebraic admissibility boundary;
  • a shock-magnitude $\alpha^*$ at which the first material arbitrage appears along each tested direction — interpretable as a distance to failure in parameter space;
  • a Durrleman $g$-surface showing where in moneyness the first arbitrage emerges;
  • a calendar-spread decrement table across adjacent tenors on a dense grid, including intermediate tenors generated under the documented front-office interpolation convention, because arbitrage between calibrated grid points may not be visible at the calibrator tenors alone (changed for clarity);
  • a Lee moment-formula compliance statement for each slice's wing slopes.

This is particularly relevant for long-dated index and single-stock surfaces ($T \geq 5$ years), where sparse strike information, extrapolated wings, and the Iijima → SVI-JWK injection for Japanese stocks can make boundary behaviour economically material. The test output is designed to be directly usable as a Finding or Action Plan input in document_2, with pre-stated acceptance criteria rather than retrospective qualitative judgment.

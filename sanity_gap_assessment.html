Sanity Check Gap Assessment — SVI-JWK document_2

Assessment of whether the Sanity Check section provides sufficient evidence of model behaviour, parameter interpretation, and implementation consistency.

Executive Summary

The Sanity Check is useful as a first-order directional control because it verifies that the volatility smile broadly moves in the expected direction when SVI-JWK parameters are perturbed. However, the test is not sufficient as a standalone validation control because it relies mainly on visual inspection, does not define numerical pass/fail thresholds, and does not fully evidence the parameter transformation chain that is central to the model change.

The most material gaps are the absence of quantitative acceptance criteria, insufficient testing of parameter cross-effects between the put and call wings, no explicit ATM anchor verification, single-date testing, and no dedicated test of the post-rescaling kurtosis-definition change across maturities. The Iijima baseline comparison is better treated as an enhancement point because comparison work is performed later in the calibration analysis, although including it earlier would have improved validation flow.

Overall assessment: The Sanity Check should be retained, but its conclusion should be qualified. It supports broad directional plausibility, not full implementation correctness or robust arbitrage-control readiness.

2. Detailed Gap Assessment

Gap 1 — No Quantitative Acceptance Criterion

The Sanity Check appears to rely primarily on visual inspection of volatility smile movements under parameter perturbations. This is useful for detecting gross implementation issues but is not sufficient to establish that the model response is quantitatively correct.

A directional response can still pass a visual check even when the magnitude of the response is materially wrong. For example, a scaling error in the treatment of ATM total variance, maturity normalisation, or wing-slope transformation may still produce a smile that moves in the expected direction, while producing economically incorrect sensitivities.

Correction to include: If νt represents ATM total variance, then a 10% increase in νt does not imply a 10% increase in ATM implied volatility. For fixed maturity, the approximate ATM implied-volatility change is about 5%.
νt = σATM2 t
σATM = √(νt / t)
ΔσATM / σATM ≈ 0.5 × Δνt / νt

Therefore, the Sanity Check should define numerical acceptance criteria for each parameter perturbation. These should include expected ATM level changes, skew changes, curvature changes, and left/right wing-slope changes, measured using explicit tolerances rather than visual judgement alone.

Recommended wording: The Sanity Check confirms broad directional behaviour but does not provide sufficient quantitative evidence to detect scaling errors, transformation errors, or unintended parameter coupling. Numerical pass/fail thresholds should be added for each parameter response.

Gap 2 — Call Wing Dependency on Put-Wing Slope

The Sanity Check notes that the call wing may be affected when the put-wing slope parameter is shifted. This should not be treated merely as a plotting observation. It is relevant because SVI-JWK parameters are intended to provide economically interpretable controls over the ATM level, skew, kurtosis, put wing, and call wing.

The point should be phrased carefully. Some degree of coupling is structurally expected because the SVI-JWK parameters are transformed into raw-SVI parameters, and the raw-SVI parameters can depend jointly on both put-wing and call-wing inputs. Therefore, the issue is not necessarily that the model is wrong. The issue is that the size and economic materiality of this coupling were not quantified in the Sanity Check.

b = √(νt t) × (ct + pt) / 2

Because the raw-SVI parameter b can depend on both ct and pt, changing the put-wing slope may influence the wider smile shape, including the call wing. This matters for trader marking, risk attribution, and sensitivity interpretation.

  • If pt materially affects the call wing, left-wing and right-wing marks are not fully independent in economic effect.
  • Risk sensitivities to pt may contain call-wing exposure, and sensitivities to ct may contain put-wing exposure.
  • This may help explain unexplained P&L effects observed in wing-risk reconciliation.
Recommended wording: The Sanity Check identified cross-dependency between wing parameters, but the effect should have been quantified. A parameter cross-sensitivity matrix should be added to measure how shifts in pt and ct affect ATM volatility, skew, left-wing slope, right-wing slope, option PV, and Greeks.

Gap 3 — No Explicit ATM Strike Verification

The Sanity Check plots smile shapes across strikes but does not appear to explicitly verify that the ATM point behaves as analytically expected. This is an important omission because the ATM point is the anchor of the SVI-JWK parameterisation.

A robust test should verify, at minimum, the following ATM properties:

  • When νt changes, ATM implied volatility should move consistently with the ATM total-variance definition.
  • When ψt changes, the ATM level should remain stable while the local slope/skew changes.
  • When pt or ct changes, the primary movement should be concentrated in the corresponding wing, subject to documented transformation coupling.
  • When κt changes, smile curvature should change without causing unintended ATM-level displacement beyond tolerance.
Recommended wording: The Sanity Check should include an ATM anchor table showing initial ATM volatility, shocked ATM volatility, expected change, observed change, tolerance, and pass/fail result for each SVI-JWK parameter perturbation.

Gap 4 — Single Market Date Tested

The Sanity Check is based on a single market date. A single observation date can confirm behaviour under one market state, but it cannot establish robustness across materially different volatility regimes.

SVI-JWK parameter behaviour may differ across high-volatility, low-volatility, steep-smile, flat-smile, and inverted-term-structure environments. This is particularly relevant for single-stock volatility surfaces and long-dated expiries, where parameter interactions may become more pronounced and less visually obvious.

Recommended wording: The Sanity Check should be repeated across multiple market regimes, for example high-volatility, low-volatility, steep-skew, flat-skew, and long-dated sparse-data environments. The current single-date test should be described as illustrative rather than conclusive.

Gap 5 — No Explicit Test of κt Redefinition Impact

This is one of the most important Sanity Check gaps. The model change includes a post-rescaling change in the definition of the kurtosis parameter. Therefore, the Sanity Check should explicitly verify that the revised κt definition behaves consistently across maturities.

Testing κt for a single expiry only verifies local curvature behaviour. It does not verify whether the revised maturity-normalised definition produces the intended behaviour at 1Y, 2Y, 5Y, and longer maturities. This is especially relevant because the observed concerns are concentrated in long-dated expiries.

Operational relevance: Ongoing monitoring that only tracks κtnew may not directly control the raw-SVI no-arbitrage condition unless the full transformation chain from κtnew to raw-SVI parameters and then to the g-function is applied consistently.
Recommended wording: The Sanity Check should include a cross-maturity κtnew test. For the same κtnew input at 1Y, 2Y, 5Y, and longer tenors, the test should evidence the transformed raw-SVI parameters, resulting smile curvature, and no-arbitrage g-function output.

Gap 6 — No Iijima Baseline Comparison in the Sanity Check

This is best classified as an enhancement rather than a standalone deficiency. Since the broader validation performs Iijima-versus-SVI-JWK comparison in the calibration analysis, it would be too strong to state that the comparison is absent from the validation overall.

However, from a validation-flow perspective, including a simple baseline comparison at the Sanity Check stage would have been useful. It would establish that SVI-JWK produces broadly consistent near-ATM and short-to-medium maturity smile behaviour in regions where the previous model was known to perform reasonably well.

Recommended wording: The absence of an Iijima comparison in the Sanity Check should be treated as an enhancement point. Adding a simple Iijima-versus-SVI-JWK overlay at the initial sanity-check stage would improve model-change traceability before moving to detailed calibration analysis.

Gap 7 — Long-Dated SoftBank/9984 Sensitivity Observation Not Fully Evidenced

The Sanity Check notes reduced visibility of put-wing and call-wing sensitivity for a long-dated SoftBank/9984 trade and attributes this to the long tenor and lower sensitivity of the smile to wing-slope changes. This explanation is plausible, but it should be supported by numerical evidence.

The concern should not be phrased as a definitive conclusion that the model does not respond to wing parameters. A more accurate validation framing is that reduced sensitivity may weaken the economic interpretability of wing marks for long-dated names and should therefore be quantified.

  • Show PV and Greek changes under controlled shocks to pt and ct.
  • Show left-wing and right-wing slope changes before and after the shock.
  • Compare the magnitude of sensitivity for short-dated versus long-dated expiries.
  • Evidence whether the reduced response is a maturity effect, a data-quality effect, or a transformation effect.
Recommended wording: The SoftBank/9984 observation should be escalated from a qualitative explanation to a targeted validation check. A numerical sensitivity table should be added for long-dated material trades to evidence whether wing parameters remain economically meaningful.

3. Proposed Wording for the Validation Document

The Sanity Check provides a useful first-order review of directional smile behaviour under SVI-JWK parameter perturbations. However, the test is primarily qualitative and visual, and therefore does not provide sufficient evidence that the implementation is quantitatively correct. The validation should define numerical acceptance criteria for each parameter shock, including the expected ATM-level response, skew response, curvature response, and wing-slope response.

In particular, where νt represents ATM total variance, the expected ATM implied-volatility response should be assessed using the relationship σATM = √(νt / t). For a fixed maturity, a small relative change in νt should translate into approximately half that relative change in ATM implied volatility. This type of quantitative benchmark is necessary to detect scaling or transformation errors that may not be visible from smile plots alone.

The Sanity Check also identifies parameter cross-effects, including the impact of put-wing slope changes on the call wing. This should be investigated quantitatively rather than treated only as a qualitative observation. Some coupling is expected because the SVI-JWK parameters are transformed into raw-SVI parameters, but the magnitude of this coupling should be measured through a cross-sensitivity matrix. This is important for trader marking, model interpretability, and risk attribution.

The validation should additionally test ATM anchoring explicitly. For each parameter perturbation, the test should report the expected ATM behaviour, observed ATM behaviour, tolerance, and pass/fail result. This would strengthen the evidence that νt, ψt, κt, pt, and ct are being applied consistently with their intended economic interpretation.

The Sanity Check should also be extended beyond a single market date. At minimum, testing should cover multiple volatility regimes, including high-volatility, low-volatility, steep-skew, flat-skew, and long-dated sparse-data environments. The current single-date test should be regarded as illustrative, not conclusive.

Finally, given the post-rescaling change in the κt definition, the validation should include a dedicated cross-maturity κtnew test. This test should evidence the full transformation from operational SVI-JWK parameters to raw-SVI parameters and then to the no-arbitrage g-function used in the validation. This is particularly important for long-dated expiries, where arbitrage risk and parameter-definition sensitivity are more material.

5. Final Assessment

The seven proposed observations are broadly correct, but they should not all be presented with the same severity. Gaps 1, 2, 3, 4, 5, and 7 are appropriate Sanity Check limitations. Gap 6 is better framed as an enhancement to validation sequencing, because the model comparison is addressed elsewhere in the validation pack.

The strongest finding is that the Sanity Check is visually persuasive but not numerically controlled. The next strongest finding is the absence of an explicit cross-maturity test for the post-rescaling κt definition, especially given the action-plan focus on long-dated 5Y+ trades and arbitrage monitoring.

Suggested conclusion: The Sanity Check should be accepted only as a preliminary plausibility check. It should not be relied upon as evidence of full implementation correctness, parameter independence, long-maturity robustness, or no-arbitrage monitoring readiness without the additional quantitative evidence described above.
